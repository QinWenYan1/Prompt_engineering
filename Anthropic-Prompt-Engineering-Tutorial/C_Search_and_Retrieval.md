## ✅ 知识点1: Claude 的搜索能力

**Claude 有强大的搜索功能...**
- Claude 可以搜索外部知识源来增强回答质量
- 支持的搜索来源包括：
  1. **维基百科**(`Wikipedia`) — 查找和检索公开文章，支持总结、综合信息、基于搜索结果创作新内容
  2. **用户自有文档** — 纯文本或向量数据库中的嵌入格式
- 搜索能力让 Claude 突破训练数据的知识边界

> 💡 **理解技巧**：搜索 = 给 Claude 装上一个「外部硬盘」——不需要把所有知识都塞进提示里
> 📋 **术语提醒**：`Search & Retrieval(搜索与检索)` = 从外部数据源获取相关信息来增强回答

## ✅ 知识点2: RAG 架构概述

**RAG是啥...**
- **RAG**(`Retrieval-Augmented Generation`，检索增强生成)是目前最主流的知识增强架构
- 核心流程：
  1. 用户提问
  2. 从向量数据库/维基百科/互联网检索相关文档
  3. 将检索结果嵌入提示作为上下文
  4. Claude 基于上下文生成回答
- RAG 的价值在于：补充 Claude 知识空白、提升回答的准确性与相关性、使用嵌入(`Embeddings`)和向量数据库工具

> 💡 **理解技巧**：RAG = 先查资料再回答问题，而不是凭记忆回答——这正是第8章「证据先行」的工程化版本
> 📋 更详细的 RAG 示例可查阅 Anthropic Cookbook

## 🔑 核心要点总结
1. Claude 支持搜索维基百科和用户自有文档
2. RAG = 检索外部数据 → 嵌入提示 → 生成回答
3. RAG 是第8章「证据先行」策略的工程实现
4. 使用嵌入(`embeddings`)和向量数据库是实现 RAG 的关键工具
